2226. Maximum Candies Allocated To K Children

C++ Code

bool possible_distribution(vector<int>& candies, long long &k, int &m) {
    long long totalChildrenServed = 0;

    // Calculate how many children can receive 'm' candies
    for (int i = 0; i < candies.size(); i++) {
        totalChildrenServed += candies[i] / m;
    }

    // If we can serve at least 'k' children, return true
    return totalChildrenServed >= k;
}

int maximumCandies(vector<int>& candies, long long k) {
    int n = candies.size();
    long long totalCandies = 0;

    // Calculate total number of candies available
    for (int i = 0; i < n; i++) {
        totalCandies += candies[i];
    }

    int s = 1;
    long long e = totalCandies / k; // Maximum possible candies per child
    int m = s + (e - s) / 2;
    int maxPossibleCandies = 0;

    // Binary search to find the maximum number of candies per child
    while (s <= e) {
        if (possible_distribution(candies, k, m)) {
            maxPossibleCandies = m;  // Update the answer
            s = m + 1; // Try for a higher value
        } else {
            e = m - 1; // Try for a lower value
        }
        m = s + (e - s) / 2;
    }

    return maxPossibleCandies;
}

Finding Maximum Candies Each Child Can Get

Problem Statement

We are given an integer array candies, where each element represents a pile of candies. We can divide a pile into multiple sub-piles but cannot merge piles together. Given an integer k (the number of children), we need to determine the maximum number of candies each child can receive while ensuring each child gets the same number of candies.

Approach

This problem is solved using Binary Search on Answer:

  • The minimum number of candies a child can receive is 1 (if possible).
  • The maximum number is sum(candies) / k (if evenly distributed).
  • We check for a possible maximum value using binary search and validate it using a helper function.

Explanation of the Approach

Binary Search Strategy

  • We define the search space between 1 and sum(candies)/k.
  • We iteratively check if it's possible to distribute m candies to each child.
  • If yes, we try to increase m, else we decrease it.
  • The final answer is stored in ans.

Helper Function

  • This function determines if it is possible to allocate m candies to at least k children.
  • It counts how many children can be served with the current pile division.

Test Cases & Explanation

Output: 5 Explanation:

Output: 0 Explanation:

  • We only have 7 candies in total.
  • We need to distribute at least 1 candy each to 11 children.
  • Since it's not possible to serve all children, the answer is 0.

  • 15 can be split into [5, 5, 5]
  • 20 can be split into [5, 5, 5, 5]
  • We now have 9 piles of 5 candies.
  • We need to distribute to 5 children, which is possible as there are 9 piles of 5 candies.
  • Thus, the maximum candies each child can receive is 5.

Time Complexity Analysis

The time complexity of this approach is O(N log M), where:

  • N is the number of elements in candies.
  • M is the sum of candies divided by k, which defines the upper bound of binary search.

In the worst case, M is approximately 10^7 (if sum of candies is large), so log M is at most 30. This results in around 10^5 * log(10^7) ≈ 10^5 * 23 = O(2.3 × 10^6), which is efficient.

Space Complexity

The space complexity is O(1), since we are using only a few extra variables (no additional storage other than the input array).

Conclusion

This approach effectively distributes candies by utilizing binary search. Instead of brute-forcing through all possibilities, we use the mathematical constraint sum(candies) / k to limit our search space, making our solution efficient even for large inputs.
